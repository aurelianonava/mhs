MSH Repo Final Project

Posted on 2017-10-19 | In JavaScript , p5js , HTML , Github

FINAL CLASS PROJECT [^1]

( ed. standards )

Project Concept:

Completion of this project is required in order to pass this class. Please be advised that your ability to complete the project, write about it, and orally present the concepts involved in your contribution is dependent on the programing concepts learned from the beginning of the course. This means that maintaining your Programming Journal will be very important.

Your Programming Journal similar to any other journal. Information is shared with you in various ways and you should reflect on what this information means, how it’s used, and how it’s applied when conditions change. Maintaining a journal requires a daily commitment to a certain amount of time dedicated to writing on your journal, specially as new information is introduced. My recommendation is to dedicate at least 10 minutes a day to your journal.

The project involves dividing a p5.js canvas into 200 x 200 rectangles. Each student will be assigned one of these rectangle areas. For instance, I’ll take the first rectangle at rect(200, 0, 200, 200); of canvas. This is the area in canvas where the object I place will be restricted to. Each student will similarly place an object in their designated 200 x 200 area and assign actions to their object. There is an area in the repository’s README.md file where each person will write about their project.

Schoology Course Group

Make sure you

  1. Respond to an email sent to you LAUSD Email account inviting you to join a Group in Schoology. In this Group, we will have the opportunity to share information, collaborate, and provide a Question and Answer place where you can post a question about a programming concept that is challenging you. Anyone can respond to this type of help post given that you have a constructive response to a posted question.

Github Project Repository

As you know, a big part of the experience of this course has been learning version control with git and Github. Github is an online environment where all kinds of people collaborate to build things, all kinds of things. Github and it’s similars are the equivalent to what Facebook is to most people.

Make sure you:

  1. Fork the mhs repository(https://github.com/aurelianonava/mhs “mhs Github Repository”).
  2. Create a branch of your forked master branch.
  3. Experiment in this branch until you commit a change or series or changes. Merge these changes to YOUR forked master. Update this forked master to the main mhs master branch that represents the main Final project.

  4. Use jsbin(jsbin.com) to experiment with your code. Once you have a desired change, copy this code to your forked repository ( to the branch you’re experimenting with.), then merge this branch to your forked master, and make a pull request to the mhs Final master.

  5. Provide your project’s Github Gist as soon as possible.

  6. Maintain ongoing project journal notes in OneNote.
  7. Use information in your OneNote Journal to explain the essential programming concepts used in your project in the mhs repository’s README.md file. The explanation should contain 1. A clear explanation of the programming concepts involved in the project; 2. Challenges encountered and how they were resolved; 3. What you learned in the process of participating in this project. 4. How you collaborated with others and how they helped you. 5. What you think about Github and how we used it in the class and in this final project.

Project Notes:

As we work on this Final Project, challenging will present themselves. Code will not work as we think it should. Lots of figuring out what went wrong will be necessary. As we break our code and attempt to fix it, we will get frustrated and even consider giving up. It is very important to remind yourself that this is normal, that it is ok to experience emotional ups and downs. What matters is an attitude of not giving up, an attitude of “I can do this, no matter how many times I get it wrong.”
