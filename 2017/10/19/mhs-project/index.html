MHS Repository Project

Project Concept:

Completion of this project is required in order to pass this class. Please be advised that your ability to complete the project, write about it, and orally present the concepts involved in your contribution is dependent on the programing concepts learned from the beginning of the course. This means that maintaining your Programming Journal will be very important.

Your Programming Journal similar to any other journal. Information is shared with you in various ways and you should reflect on what this information means, how it’s used, and how it’s applied when conditions change. Maintaining a journal requires a daily commitment to a certain amount of time dedicated to writing on your journal, specially as new information is introduced. My recomendation is to dedicate at least 10 minutes a day to your journal.

The project involves dividing a p5.js canvas into 200 x 200 rectangles. Each student will be assigned one of these rectangle areas. For instance, I’ll take the first rectangle at rect(200, 0, 200, 200); of canvas. This is the area in canvas where the object I place will be restricted to. Each student will similarly place an object in their designated 200 x 200 area and assign actions to their object. There is an area in the repository’s README.md file where each person will write about their project.

Github Project Repository

As you know, a big part of the experience of this course has been learning version control with git and Github. Github is an online environment where all kinds of people collaborate to build things, all kinds of things. Github and it’s similars are the equivalent to what Facebook is to most people.

Make sure you:

  1. Fork the mhs repository.
    1. Use jsbin to experiment with your code.
    2. Provide your project’s Github Gist as soon as possible.
    3. Maintain ongoing project journal notes in OneNote.
    4. Use information in your OneNote Journal to explain the essential programming concepts used in your project in the mhs repository’s README.md file. The explanation should contain 1. A clear explanation of the programming concepts involved in the project; 2. Challenges encountered and how they were resolved; 3. What you learned in the process of participating in this project. 4. How you collaborated with others and how they helped you. 5. What you think about Github and how we used it in the class and in this final project.
