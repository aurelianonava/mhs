So, again, when the if()test is false, the function draw(); continues to runs. When the if()test is true, the code inside the { }curly braces runs. The computer determines whether the test is true or false by evaluating the expression inside the parentheses.

We have at present an if() condition that loops the point for a reason. It is important for you to understand what makes this effect. Note: Each time x is reset back to 0, the test inside if() continues to be false ( not pass ), therefore the function draw runs over and over again, executing if() over and over again. The test in if() becomes true ( passes ) only for an instance, when it reaches 241, and when it does, it allows the statement inside the { } curly braces to run, resenting x to x = 0, which in turn makes the point move again until it reaches 241 again, making the if() test pass again, and again.

If() rule:

if (condition) {
block of code to be executed if the condition is true
}

The else statement:

if (condition) {
block of code to be executed if the condition is true
} else {
block of code to be executed if the condition is false
}

New Tools

Take a look at these system variables, [width] and [height] on the p5.js documentation.

How can we put to use these two system variables? As the documentation explains, these two system variables store the width and height of canvas created by the two parameters in createCanvas( ). The values you placed in each parameter are stored in width and height.

What is this doing? How would you read it and explain it?

if( x == width ) {
x = 10;
}

Keep in mind that a single = in programming represents assignment, not equality. For equality the double == is used.

For example: x = 10 means “Assign ( store or save ) to variable x the integer 10.” Whereas x == 10 means “Variable x equals integer 10.”
