Using External Files To Manage Code

Before we start with this project we need to know how to manage our code with external files. When we do this it allows us to manage our code better as it grows.

Pay attention to the different JSbin tabs that you can select and deselect to provided different views. We will be working mostly with the html, JavaScript, and Output tabs.

Examine the <script src="script.js"></script> element below: 

<!DOCTYPE html>
<html>
<head>
  <title>Our Starting Point: Dot</title>
  <script data-require="p5.js@*" data-semver="0.5.7" src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/0.5.7/p5.min.js"></script>
  <link rel="stylesheet" href="style.css" />
  <script src="script.js"></script>
</head>
<body>
  <h1>Our Starting Point: Dot</h1>

  </body>
</html>

The code should run the same as when we had the functions embedded inside the index.html page. From here on we’ll be using external files such as this script.js file to manage our p5.js code.

Let’s have some fun with point();

On script.js, type the code below to create. 

function setup() {
createCanvas(480, 120);
background(200);
}
function draw() {
 strokeWeight(3);
point(5, 60);
}

Point at x = 5; y = 60.

All we have at the moment is a point at (5, 60). Let’s do something cool now. Let’s make our point move along the x axis of canvas. You created a series of points in Project 1, so you know that in order to do this you had to change the values in function point(x,y); for x and y. These variables inside the function parenthesis are called parameters. Please remember this.

Function point(x, y); has two parameters, x and y, and Parameters receive information as arguments. When we used point() we used integers instead of a variable x and a variable y. Now we will use variables inside point(x, y) and learn ho two pass on values ( like numbers ) and arguments. To do this, we are going to create some variables.

Declare ( to create ) two variables outside the functions, name them x and y, , and initialize them to x = 10, and y = 60; 

var x;    variable x declared 
var y;    variable y declared
x = 10;   // variable x initialized at 10
y = 60;  // variable y initialized at 60

function setup() {
  createCanvas(480, 120);
  background(200); 
}

function draw() {
   strokeWeight(3);
   point(x, y);
}

Run program and see what you get. Nothing changed from the last run. Why?

We have our point at (10, 60). Notice where point(x, y); is getting the arguments representing x = 10, and y = 60 from?

We will now learn how to increment the value of x inside point(x, y); with each iteration of the draw(); function. Remember that function draw() executes ( runs ) at 60 times per second.

To increment ( increase gradually ) the x argument inside point(x, y) we will use a special operator called compound operator.

These are the basic assignment operators we will be using:

Let’s put to use the addition assignment operator as follows, inside point(x+=1, y);:

function draw() {
strokeWeight(3);
point(x+=1, y);
}

And the effects is:
Incrementing point along x with x+=1

Alright, good, but we really don’t want that line drawn all across like it’s drawing it. How do we remove this effect from happening? Think about what is happening and why? What’s the solution?

After you figure it out, you should have the desired effect.
Point moving along the x axis of canvas.

As we can see, the point is moving, and moving, without stopping. It moves even beyond the present width of canvas, which is 480 right now. We will now learn how to make it stop at a designated place in canvas.
