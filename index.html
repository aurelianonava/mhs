Conditional Statements

When the test, what is inside the parentheses, passes–it becomes true–then the statement between the curly braces will run:

if (test)
statements
}

For example, in the following code, the point will continue to move along the x axis as long as test inside the if( test ) does NOT pass. 

function draw() {
strokeWeight(3);
point(x+=1, 25);
if (test) {
  statements
}
}

Once the test inside if( test ) passes, the statement that is inside the { } curly braces runs.

Take a look at this diagram:

Again, until the test does NOT pass, according to the diagram, function draw() continues to run.

Let’s setup a test. 

function draw() {
strokeWeight(3);
point(x+=1, 25);
if ( x > 240 ) {
  x = 0;    }
}

What will happen here? The point will move along the x axis until it reaches x = 240, which makes that test false ( does not pass: 240 > 240 is NOT true. ), therefore the statement inside the { } curly braces executes ( runs ) and x is repositioned at x = 0.

Try it out.

See me here

So, again, when the if()test is false, the function draw(); continues to runs. When the if()test is true, the code inside the { }curly braces runs. The computer determines whether the test is true or false by evaluating the expression inside the parentheses.

We have at present an if() condition that loops the point for a reason. It is important for you to understand what makes this effect. Note: Each time x is reset back to 0, the test inside if() continues to be false ( not pass ), therefore the function draw runs over and over again, executing if() over and over again. The test in if() becomes true ( passes ) only for an instance, when it reaches 241, and when it does, it allows the statement inside the { } curly braces to run, resenting x to x = 0, which in turn makes the point move again until it reaches 241 again, making the if() test pass again, and again.

If() rule:

if (condition)
block of code to be executed if the condition is true
}

The else statement:

if (condition)
block of code to be executed if the condition is true
} else
block of code to be executed if the condition is false
}

For example: x = 10 means “Assign ( store or save ) to variable x the integer 10.” Whereas x == 10 means “Variable x equals integer 10.”

All we have at the moment is a point at (5, 60). Let’s do something cool now. Let’s make our point move along the x axis of canvas. You created a series of points in Project 1, so you know that in order to do this you had to change the values in function point(x,y); for x and y. These variables inside the function parenthesis are called parameters. Please remember this.

Function point(x, y); has two parameters, x and y, and Parameters receive information as arguments. When we used point() we used integers instead of a variable x and a variable y. Now we will use variables inside point(x, y) and learn ho two pass on values ( like numbers ) and arguments. To do this, we are going to create some variables.

Declare ( to create ) two variables outside the functions, name them x and y, , and initialize them to x = 10, and y = 60; 

var x;    variable x declared 
var y;    variable y declared
x = 10;   // variable x initialized at 10
y = 60;  // variable y initialized at 60

function setup() {
  createCanvas(480, 120);
  background(200); 
}

function draw() {
   strokeWeight(3);
   point(x, y);
}

Run program and see what you get. Nothing changed from the last run. Why?

We have our point at (10, 60). Notice where point(x, y); is getting the arguments representing x = 10, and y = 60 from?

We will now learn how to increment the value of x inside point(x, y); with each iteration of the draw(); function. Remember that function draw() executes ( runs ) at 60 times per second.

To increment ( increase gradually ) the x argument inside point(x, y) we will use a special operator called compound operator.

Examples and Reference

Learning how to program with p5.js involves exploring lots of code: running, altering, breaking, and enhancing it until you have reshaped it into something new. With this in mind, the p5.js website has dozens of examples that demonstrate different features of the library. Visit the Examples page to see them. You can play with them by editing the code of each example on the page and clicking “run.” The examples are grouped into categories based on their function, such as Form, Color, and Image. Find an interesting topic in the list and try an example.

If you see a part of the program you’re unfamiliar with or want to learn more about its functionality, visit the p5.js Reference.

The p5.js Reference explains every code element with a description and examples. The Reference programs are much shorter (usually four or five lines) and easier to follow than the examples on the Learn page. Note that these examples often omit setup() and draw() for simplicity, but the lines you see there are intended to be put inside one of these blocks in order to run.
